# Threat model distinguishes default guarantees from optional hardening

Canopy's default threat model protects against unauthorized public observers, reduces trust in curious hosting servers through encryption and signatures, provides forward revocation for revoked collaborators, and limits compromised agents, CI jobs, or tools through derived purpose-scoped capabilities. Canopy does not pretend to solve malicious authorized maintainers, traffic analysis, or compromised local devices by default; those require governance, metadata hardening, client hardening, or operational controls. Maintainers may govern policy but cannot bypass cryptographic access to past private data without capabilities, and sensitive maintainer actions are governance events; multi-party governance is available as optional hardening.
